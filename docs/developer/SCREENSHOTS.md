# Maintaining documentation screenshots

Screenshots in `docs/screenshots/` are generated by Playwright and committed.

**Automated (GitHub Actions):** trigger the
[Update Documentation Screenshots](https://github.com/matthewhand/chatty-commander/actions/workflows/screenshots.yml)
workflow from the Actions tab. It also runs on pushes to `main` that touch the
frontend or backend routes.

**Manual (local):**

```bash
cd webui/frontend
npm run generate-docs        # runs tests/e2e/screenshots.spec.ts
npx playwright test tests/e2e/guided_tour.spec.ts   # regenerates the tour-XX images
```

Both spec files auto-start the backend (`--web --test-mode --no-auth` on port
8100) via the Playwright `webServer` config and save 1280x800 captures to
`docs/screenshots/`.
