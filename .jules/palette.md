## 2024-05-24 - DaisyUI `form-control` Label Linking
**Learning:** DaisyUI's `form-control` component often visually groups labels and inputs, but developers frequently forget to explicitly link them using `htmlFor` and `id` attributes. This breaks accessibility, especially for screen readers and click-to-focus behavior. For inputs like textareas that lack visual labels by design, an explicitly linked label with the `sr-only` class is essential to provide context without compromising the UI.
**Action:** When auditing or implementing forms, especially when using utility classes or component libraries like DaisyUI, always ensure that every `<input>`, `<select>`, and `<textarea>` has a unique `id` and an associated `<label>` with a matching `htmlFor` attribute. Use `sr-only` labels when visual labels are not desired.
