## 2024-05-30 - Fix Path Traversal Authentication Bypass
**Vulnerability:** The authentication middleware was using `path.startswith("/api/")` to protect endpoints and checking `path.startswith()` for public endpoints, but it did not normalize paths, allowing a path traversal bypass like `/docs/../api/secret`. Additionally, it allowed partial matches like `/docs-bypassed` to pass as public endpoints.
**Learning:** ASGI applications (like FastAPI/Starlette) do not always pre-normalize `request.url.path` before custom middleware is called. The `.startswith()` string method is insecure for checking URL paths because it does not understand directory boundaries and doesn't handle `..` components.
**Prevention:** Always normalize incoming HTTP paths using `posixpath.normpath(request.url.path)` before access control checks. Always use exact path matching (`path == endpoint`) or strict directory boundary checks (`path.startswith(endpoint + "/")`) when verifying route access.

## 2024-05-31 - Fix API Key Timing Attack Vulnerability
**Vulnerability:** The authentication middleware was verifying the `X-API-Key` header by comparing the provided key against the expected key using a standard string equality operator (`api_key != expected_key`).
**Learning:** Standard string comparisons evaluate character by character and return `False` as soon as a mismatch is found. This creates a timing side-channel where an attacker can measure the time taken to reject an invalid key. By iterating through possible characters and monitoring the response time, the attacker can systematically deduce the correct API key, bypassing authentication entirely.
**Prevention:** Always use constant-time comparison functions, such as `secrets.compare_digest(api_key, expected_key)`, when validating sensitive credentials like API keys, tokens, passwords, or cryptographic hashes. Since `compare_digest` raises a `TypeError` if either argument is `None`, ensure explicit null/empty checks (`if not api_key:`) are performed before the constant-time comparison. Additionally, always encode the strings to bytes (`.encode('utf-8')`) before comparison to prevent `TypeError` exceptions if the keys contain non-ASCII characters.
