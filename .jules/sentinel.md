## 2024-05-30 - Fix Path Traversal Authentication Bypass
**Vulnerability:** The authentication middleware was using `path.startswith("/api/")` to protect endpoints and checking `path.startswith()` for public endpoints, but it did not normalize paths, allowing a path traversal bypass like `/docs/../api/secret`. Additionally, it allowed partial matches like `/docs-bypassed` to pass as public endpoints.
**Learning:** ASGI applications (like FastAPI/Starlette) do not always pre-normalize `request.url.path` before custom middleware is called. The `.startswith()` string method is insecure for checking URL paths because it does not understand directory boundaries and doesn't handle `..` components.
**Prevention:** Always normalize incoming HTTP paths using `posixpath.normpath(request.url.path)` before access control checks. Always use exact path matching (`path == endpoint`) or strict directory boundary checks (`path.startswith(endpoint + "/")`) when verifying route access.

## 2024-05-30 - Fix Memory Exhaustion DoS in RateLimitMiddleware
**Vulnerability:** The `RateLimitMiddleware` implementation used a `collections.defaultdict(list)` to track request timestamps per client IP. Although the list sizes were limited per IP, the dictionary keys (IP addresses) were never explicitly deleted when old timestamps expired. This created a Memory Exhaustion Denial of Service (DoS) vulnerability. An attacker making requests from many different IP addresses (or spoofing IPs if trusted proxies weren't properly configured) could cause the `self.requests` dictionary to grow infinitely, eventually consuming all available server memory and crashing the application.
**Learning:** In stateful tracking mechanisms (like rate limiting, connection tracking, or caching) that store data per user or IP, simply dropping old items from inner collections is insufficient. The parent data structure must also be periodically pruned of inactive keys to prevent unbounded memory growth.
**Prevention:** Implement a global cleanup mechanism (e.g., `_last_cleanup`) that periodically iterates over the tracking dictionary and safely removes expired entries and empty keys.
